# Backup restore fixtures

GymTrack keeps committed backup fixture resources under `app/src/androidTest/assets/backup-fixtures/` so backup compatibility can be tested against stable historical samples instead of only archives generated by the current code.

## schema8-v1

- Source backup format: 1
- Source app version: 0.9.0-schema8-fixture
- Source Room schema version: 8
- Purpose: representative pre-current-schema full backup restored into the current app schema.

Coverage:

- settings with multiple categories
- legacy notes with category, text, ordering, and learnings
- legacy exercises, aliases, and sets
- canonical categories, exercises, aliases, workouts, workout exercises, and workout sets
- mixed stored weight units, including KG and LB
- canonical parent exercise ordering
- history/editor-visible text after restore
- statistics-visible legacy set data after restore
- selected-workout CSV export-visible data after restore

## schema8-format99

This fixture reuses the same payload with an unsupported manifest format version. It verifies that unsupported backup versions fail before restore instead of being partially imported or guessed.

## Maintenance policy

- Do not mutate existing fixture files casually. Add a new fixture directory when compatibility behavior changes.
- If data.json changes, recompute and update manifest.json payloadSha256 and record counts.
- If backup JSON field names, required entries, checksum rules, or restore semantics change, add either a compatibility reader or a new unsupported-version rejection fixture.
